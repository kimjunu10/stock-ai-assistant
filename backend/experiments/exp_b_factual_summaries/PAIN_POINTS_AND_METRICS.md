# EXP-5 사실 통합 본문 — 핵심 Pain Point & 정밀 통계

> 단계: 확정 클러스터링 재사용 → Solar Pro 라벨 비의존 사실 통합 본문 생성 (오프라인 산출물).
> 실행 환경: 로컬 Mac(MPS) 임베딩 + Upstage Solar Pro3 API. Supabase 미변경.

## 1. 이 작업이 풀려던 실무 핵심 Pain Point

| # | Pain Point | 왜 문제였나 | 이 단계의 해결 |
|---|---|---|---|
| P1 | **중복 뉴스 폭증** | 같은 사건을 수십~수백 언론사가 중복 보도. 기사 단위로 보여주면 사용자가 같은 내용을 반복해서 봄 | 종목별·시간순 online centroid 로 사건 클러스터링 → **7,735건 → 1,298 클러스터 (6.0:1 압축)** |
| P2 | **기사별 LLM 호출 비용** | 기사마다 요약·분류하면 LLM 호출이 기사 수만큼 발생 | 사건 단위로 묶어 **클러스터당 1회**만 LLM 요약 → 호출 6배 절감 |
| P3 | **통합 본문의 사실 왜곡 위험** | LLM이 여러 기사를 합치며 없는 사실을 지어내거나(환각) 투자 판단(호재/악재)을 주입할 수 있음 | 프롬프트에 "원문 사실만 보존, 새 사실·판단 표현 금지" 강제 + JSON 스키마 강제. **원문(articles)은 일절 미변경** |
| P4 | **over-merge (서로 다른 사건 뭉침)** | 시장 전체 뉴스("코스피 급락")가 종목 사건에 섞여 통합 본문이 부정확해짐 | 확정 설정 유지(지시 준수)하되 **진단 정보 산출 + 별도 분석 문서**로 식별·정량화 → 후속 완화책 권고 |
| P5 | **감성 라벨 오염 방지** | gold label 을 자동 생성하거나 모델 예측을 정답으로 쓰면 이후 감성 실험이 무의미 | `gold_label`/`label_reason` 을 **빈 칸으로 남긴** reference template 만 생성하고 STOP |
| P6 | **재현성** | 모델 버전·전처리가 바뀌면 결과가 달라져 비교 불가 | 임베딩/Solar 모델 revision, 전처리 버전, git commit, 프롬프트 전문을 `summary_run_env.json` 에 고정 |

## 2. 정밀 통계

### 2-1. 입력 데이터
- relevant (기사, 종목) 연결: **7,735건** (실행 시점, 30분 배치로 계속 증가 중)
- 종목: 삼성전자·SK하이닉스·현대차·한화오션·두산에너빌리티 (5개)
- description 보유율 100%, body 보유율 ~98%

## 3. 알려진 한계 (정직하게)

1. **over-merge**: 50+ 클러스터 34개 중 ~15개가 시장 전체 뉴스 뭉침 의심. 상세는 `OVER_MERGE_ANALYSIS.md`.
2. **초대형 클러스터 요약 대표성**: 클러스터당 앞 12건만 Solar 입력 → 353건짜리는 전체를 대표 못 함(토큰 폭발 방지 트레이드오프).
3. **재시도 토큰 추정치**: 정확 집계가 아닌 평균 기반 근사(§2-5 주석).
4. **시간순 holdout 아님**: 현재 축적분 전체 배치 재현. 운영 검증은 shadow test 필요.
