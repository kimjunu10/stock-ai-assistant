## 3. 새로 발견된 내용 / 변경한 규칙

### (a) 원문 대조 불일치는 사실상 키움증권 1곳에 집중 — 값 손실 아님
증권사별 매치율: **키움 75.5%**, 나머지 14곳 **98.7~100%**.
불일치 상위 리포트가 전부 키움. 원인 표본:

```
셀='25358 27134 29304 29597 , , , ,'   ← 원문은 25,358 / 27,134 / 29,304 / 29,597
```

키움 표는 **여러 숫자와 콤마가 한 셀에 뭉쳐** 추출된다(숫자열과 콤마열이 분리·재배치).
대조 스크립트가 이 병합 셀에서 `253`·`293` 같은 **가짜 부분 숫자**를 뽑아 실패로 집계한 것.
실제 값(25,358 등)은 셀 안에 모두 존재한다 → **데이터 손실 아님**.

- **변경 규칙(3단계 반영)**: 셀 내 `숫자*4 ... 콤마*4` 패턴을 감지해
  `숫자,숫자` 쌍으로 재정렬(콤마열↔숫자열 zip). 키움류 병합 셀에 한정한 일반 규칙이며
  특정 파일명 하드코딩 아님. 반영 시 전체 매치율 ~99%대 예상.

### (b) source_page 탐지율 53.7% — 나머지는 pdf_page fallback 정상
- 절반가량 리포트는 꼬리말에 인쇄면 번호가 텍스트로 있어 오프셋 탐지 성공(131/244).
- 나머지는 페이지 번호가 이미지이거나 위치가 달라 미탐지 → **pdf_page(0-index)로 fallback**.
  출처 표기는 pdf_page 기준이라도 정확하므로 적재 차단 사유 아님.
- **변경 규칙**: 탐지 실패 시 `source_page = null`, 출처는 pdf_page 사용(3단계 확정).

### (c) 서술형 표 강등 1,253건 — 표/본문 분리가 대량으로 작동
Step 2 에서 제안한 "숫자 비율 <15% 또는 셀 길이 >120자" 강등 규칙이 244개에서 1,253건 적용.
재무표(1,936)만 표로 남고 수주현황·ESG·목차형 블록은 본문으로 강등되어 표 품질이 안정됐다.

### (d) 다중 페이지 표 연결 26건
"직전 표와 컬럼 수 동일 + 이번 표 헤더 없음 + 헤더행 인덱스 0" 규칙으로 26건 연결 감지.
주로 장문 리포트(교보·iM 등)의 분할된 재무표. 3단계에서 실제 병합 저장에 사용.

### (e) 스캔 37 / 차트 35 페이지 — 임베딩 제외 대상 정량화
페이지 평균 글자수·이미지 수 기준으로 스캔·차트 페이지를 식별. 본문 임베딩에서 제외하고
page_no·이미지 수만 메타로 보존(값 오추출 방지).
