### 1-2. 파일명 규칙
형식: `YYYY-MM-DD_종목_증권사_제목.pdf`
정규식 `^(\d{4}-\d{2}-\d{2})_([^_]+)_([^_]+)_(.+)$` 로 발행일·종목·증권사·제목 100% 분해.
→ report_date / broker / title 메타데이터를 파일명만으로 확보 가능(본문 재확인은 보강용).

### 1-3. ⚠️ 중요 발견: macOS NFD 정규화
파일시스템이 폴더·파일명을 **NFD(자모 분리)** 로 반환한다.
파이썬 dict 리터럴(NFC)로 매칭하면 실패하므로, **파서는 모든 파일시스템 문자열을
`unicodedata.normalize("NFC", ...)` 처리 후 DB·정규식과 대조해야 한다.**
(본 조사에서 dryrun 스크립트는 NFC 매칭으로 14/14 파일 경로 해석 성공을 확인함.)

## 2. 대표 PDF 14개 시험 파싱 결과

증권사 9곳 · 종목 5개 · 요구 유형(텍스트중심/재무표중심/차트중심/다중페이지/A·E·F/추출난도高) 전부 포함.

### 2-1. 잘 되는 것
- **본문 텍스트**: 스캔 1건 제외 전부 페이지당 수천 자, `get_text("text")` 로 충분.
- **핵심 메타데이터**: 투자의견(매수/BUY/보유/중립), 목표주가, 종목코드가 1~2p에서 안정적으로 추출.
- **A/E/F 구분**: `2025A`(실제) / `2026E`(추정) / `2027F`(전망) / `2026P`(잠정) 패턴이
  증권사 불문 일관. 정규식 `(20\d{2})\s*\.?\s*([AEFP])` 로 연도별 값 종류 판별 가능
  → `research_report_tables.value_kind` 채우기 가능.
- **페이지 출처**: 모든 텍스트·표 청크에 page_no 보존.

### 2-2. 문제점 (일반화 규칙 필요, 특정 증권사 하드코딩 금지)
1. **표 셀 병합/컬럼 붕괴** — `find_tables().extract()` 가 1p 상단 재무 요약표에서
   숫자를 한 셀에 뭉치거나 빈 컬럼을 다수 생성(예: iM증권 8x22 표, 미래에셋 6열 표).
   → 재무 요약표는 좌표 기반 컬럼 재정렬 또는 `strategy="lines"/"text"` 옵션 비교 필요.
2. **표지 요약표가 텍스트 덩어리로 병합** — 대신증권 리포트에서 제목+본문+지표가 한 셀에.
   → 1p는 "요약 헤더 영역"으로 별도 처리(제목/의견/목표가 규칙 추출) 권장.
3. **차트 축 라벨이 제목으로 오인** — 폰트 크기 최댓값만으로 제목 추정 시
   `80 130 180` 같은 축 눈금이 잡힘.
   → 제목은 (a) 파일명 title + (b) 1p 상단 y좌표 영역 + (c) 숫자-only 제외 규칙 결합.
4. **스캔/이미지 리포트 1건** — 텍스트 64자. OCR 없이는 본문 검색 불가.
   → parse_status='partial'/'failed' 로 표시하고 발표 대상에서 제외(스펙 §Phase 5 정책과 일치).
5. **차트 중심 페이지**(drawing 수천 개)는 표·텍스트가 빈약 → 페이지 단위로 "본문 없음"
   판정 후 스킵하되 page_no·이미지 개수는 메타로 보존.

## 3. 추천 파서 구조 (다음 단계 설계안, 구현 아님)

```
parse_report(pdf_path):
  1) 파일명 → (report_date, stock_name, broker, title)  # NFC 정규화 필수
  2) stock_name → stock_code (stocks 테이블 조회)
  3) file_hash(sha256) → research_reports.file_hash 중복 스킵
  4) fitz.open → 암호화/손상/스캔(평균 글자수<80) 판정 → parse_status
  5) 페이지 루프:
       - 본문: get_text("text") + 좌표 blocks 로 단락 복원 → research_report_pages(page_no, text)
       - 표: find_tables().extract() → 정제(빈 컬럼 병합, NFC) → research_report_tables
             · 헤더행에서 연도+A/E/F 파싱 → value_kind(actual/estimate/forecast)
       - 모든 청크에 page_no 부착(출처 보존)
  6) 1p 헤더 영역 규칙: 투자의견/목표주가/현재주가 추출 → research_reports 필드
  7) 임베딩 대상: 본문 단락 + 표 캡션/셀 텍스트 (passage 모델)
```

- 특정 증권사·파일명 하드코딩 없음. 레이아웃 차이는 아래 일반화 분류로 흡수:
  - **Type A (표준 애널리스트 리포트)**: 1p 헤더+요약표, 2p~ 본문·재무표. 다수.
  - **Type B (단문/코멘트)**: 2~4p, A/E/F 적음(Spot/Conference call). 규칙 동일, 표 적음.
  - **Type C (차트 중심)**: drawing 수천, 본문·표 빈약 → 페이지 스킵 규칙.
  - **Type D (스캔/이미지)**: 텍스트 거의 없음 → partial/failed, 발표 제외.
