## 2. 검증 항목 1~12 결과

1. **본문 순서 복원** — 좌표 blocks 를 (컬럼, y) 정렬해 2단 레이아웃 순서 복원. OK.
2. **제목·본문·표·머리말·꼬리말 분리** — 상/하단 6% 영역 + 페이지번호/면책 후보를
   머리말·꼬리말로 제거(리포트당 평균 수 블록). 제목은 1p 최대폰트 span(숫자 제외). OK.
3. **표 행·열 보존** — `find_tables().extract()` 로 행·열 유지. 표준 재무표는 양호.
4. **표 제목·열 머리글 연결** — 헤더행을 연도/A·E·F 토큰 최다 행으로 자동 선택. OK.
5. **연도·분기·단위·통화 보존** — 헤더에서 `2025A/2026E/4Q26F` + 단위(십억원·억원·원·%·배·주)
   추출. OK(단위는 표별 감지).
6. **A/E/F → value_kind** — `A→actual, E→estimate, F→forecast, P→estimate,
   가이던스 신호→guidance, 연도만/판별불가→unknown`. 50/50 표에서 성공.
7. **다중 페이지 표 연결** — 페이지 경계 넘는 표는 이번 단계에서 개별 표로 추출됨(연결 미구현).
   → 3단계에서 "직전 페이지 표 헤더와 동일 컬럼 수 + 헤더 없는 연속 표" 규칙으로 연결 제안.
8. **축라벨·각주·페이지번호 혼입 여부** — 축 라벨(숫자-only 짧은 블록)·페이지번호·면책은
   본문에서 제거. 다만 **차트 이미지 안 숫자**는 애초에 텍스트로 안 잡혀 표/본문 혼입 없음.
9. **숫자·부호·소수·% 손실** — 값 자체 손실은 없음. 불일치 69건은 손실이 아니라
   (a) 소수점/콤마 토큰 분리("55 ." ← "55.0", "333614 ," ← "333,614"),
   (b) 서술형 텍스트 블록을 표로 오인 — 아래 §3 참조.
10. **source_page·pdf_page 보존** — pdf_page(0-index) 전량 보존. 인쇄면 번호(source_page)는
    리포트마다 위치가 달라 2단계에선 미확정(3단계에서 리포트별 규칙 필요).
11. **추출값 원문 대조** — 표 셀 숫자를 페이지 원문 텍스트와 대조. 97.6% 일치.
12. **하드코딩 없음** — 증권사/파일명 분기 없이 정규식·좌표·폰트 일반 규칙만 사용.

## 4. 실패·문제 원인과 일반화 가능한 수정 규칙

### (a) 원문 대조 불일치 69건 (값 손실 아님)
- **원인 1**: 표 셀에서 소수점/천단위 콤마가 별도 토큰으로 분리("55 ." ← "55.0",
  "333614 ," ← "333,614"). 값은 보존되나 대조 문자열이 어긋남.
  → **수정 규칙**: 셀 정규화 시 `숫자 뒤 공백+구분자(.,)` 를 붙여 재조립.
- **원인 2**: 서술형 텍스트 블록(수주 현황·ESG 표 등)을 `find_tables` 가 표로 오인.
  → **수정 규칙**: 표 후보 중 "숫자 셀 비율 < 15%" 이거나 "한 셀 길이 > 120자" 인 표는
  본문 블록으로 강등(재무표만 표로 유지).

### (b) 스캔/이미지 리포트 1건 (미래에셋 SK하이닉스 NAND)
- 본문 56자/7p → 텍스트 없음.
  → **수정 규칙**: 페이지 평균 글자수 < 80 이면 `parse_status='partial'`,
  전 페이지가 그러면 `'failed'` 로 표시하고 발표 대상에서 제외(스펙 §Phase 5 정책과 일치).

### (c) 차트 중심 페이지
- drawing 수천 개, 값은 차트 이미지 안 → 텍스트로 안 잡힘(표/본문 혼입 없음, 안전).
  → **수정 규칙**: 페이지 본문 블록 < 3 이고 이미지 다수면 "차트 페이지"로 표시,
  page_no·이미지 개수만 메타로 보존하고 임베딩 대상에서 제외.

### (d) 다중 페이지 표 연결 (미구현)
  → **수정 규칙(3단계)**: 직전 페이지 마지막 표와 컬럼 수가 같고 헤더행이 없는 후속 표는
  같은 표의 연속으로 병합.

### (e) source_page(인쇄면) 미확정
  → **수정 규칙(3단계)**: 리포트 꼬리말/특정 위치의 페이지 번호를 리포트별로 1회 탐지해
  pdf_page ↔ source_page 오프셋을 잡되, 실패 시 pdf_page 만 사용.

## 6. 추천 파서 구조 (3단계 이후, 구현 아님)

```
parse_report(pdf):
  1) 파일명 → (date, broker, title)  [NFC 정규화]
  2) folder→stock_code (stocks 조회)
  3) file_hash 중복 스킵
  4) open → 암호화/스캔(평균<80자) 판정 → parse_status(success/partial/failed)
  5) 페이지 루프:
       본문: 좌표 blocks 2단 정렬 + 머리말/꼬리말/축라벨 제거 → pages(page_no, text)
       표:  find_tables → [셀 정규화(소수·콤마 재조립)] → [서술형 표 강등 규칙]
            → 헤더 연도/A·E·F → value_kind(열별) → tables(page_no, rows, value_kind)
       차트 페이지(본문<3+이미지多)는 메타만 보존, 임베딩 제외
  6) 다중 페이지 표 병합(컬럼 수 동일+헤더 없음)
  7) 1p 헤더에서 투자의견/목표주가/현재주가 추출
  8) 임베딩 대상: 본문 단락 + 표 캡션/헤더 (passage) — [3단계에서만]
```

value_kind: `actual / estimate / forecast / guidance / unknown` (본 드라이런에서 검증).

## 7. 전체 적재 전 수정이 필요한 문제 (요약)

1. 표 셀 소수점·콤마 정규화(원문 대조 불일치 해소).
2. 서술형 텍스트 블록의 표 오인 강등 규칙.
3. 스캔/차트 페이지 partial/제외 정책 적용.
4. 다중 페이지 표 연결 규칙.
5. source_page(인쇄면) 오프셋 탐지(실패 시 pdf_page fallback).

위 5개는 전부 §4의 일반 규칙으로 처리 가능하며, 특정 증권사·파일명 하드코딩 불필요.
