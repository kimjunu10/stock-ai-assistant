# 뉴스 처리·클러스터링 운영 기준

이 문서는 현재 활성 뉴스 v2 파이프라인의 기준을 정리한다. 과거 `experiments/` 아래
산출물의 설정과 충돌할 때는 이 문서와 실제 운영 코드를 우선한다.

## 전체 처리 순서

1. 뉴스를 수집하고 본문을 크롤링한다.
2. 기사와 종목의 직접 관련성을 판정한다.
3. 관련 기사 중 실제 기업 사건으로 묶을 수 있는 `company_event`를 분류하고
   `event_signature`를 만든다.
4. BGE-M3로 같은 종목의 유사 사건 후보를 찾는다.
5. Solar가 후보와 새 기사가 실제로 같은 사건인지 최종 판정한다.
6. 신규 클러스터를 만들거나 기존 클러스터에 기사를 추가한다.
7. Solar로 사건 전체의 제목·쉬운 설명·사실 본문을 생성해 저장한다.
8. 확정된 클러스터 제목으로 감성을 분류해 저장한다.
9. API가 클러스터·요약·감성 결과를 반환한다.

## 역할 분류

모든 관련 기사를 무조건 기업 사건 클러스터에 넣지 않는다. 기사 역할 분류에서
`company_event`이면서 `event_eligible=true`인 기사·종목 pair만 v2 사건
클러스터링 대상으로 사용한다.

역할 분류 결과에는 주체, 행동, 대상, 제품·프로젝트, 금액, 사건 날짜, 고유 식별자를
담은 `event_signature`가 포함된다. 이 값은 최종 동일 사건 판정의 보조 근거다.

## BGE-M3 후보 검색

- 모델: `BAAI/bge-m3`
- revision: `5617a9f61b028005a4858fdac845db406aefb181`
- 입력: 기사 `title + description`
- 차원: 1024
- 범위: 같은 `stock_code`
- 활성 후보 창: 최근 24시간
- 후보 최소 cosine similarity: 0.55
- 최대 후보 수: 5개

BGE-M3 유사도는 최종 병합 결과가 아니다. 표현과 주제가 비슷한 후보를 빠르게
찾아 Solar에 넘기는 1차 검색 값이다. 유사도가 높더라도 서로 다른 행사나 발표라면
Solar가 새 클러스터로 분리해야 한다.

과거 오프라인 실험에서 사용한 72시간 창은 현재 운영 기준이 아니다.

## Solar 동일 사건 판정

Solar에는 새 기사의 제목·설명·`event_signature`와 후보별 최초 기사, 대표 기사,
최근 기사, 기존 `event_signature`를 전달한다.

다음 요소를 비교해 실제로 같은 발표·발언·계약·행사인지 판단한다.

- 주체: 누가 발표하거나 결정했는가
- 행동: 무엇을 했는가
- 대상: 어떤 제품·정책·프로젝트·상대방에 대한 것인가
- 사건 정체성: 행사명, 주최·초청 주체, 행사 형태, 장소와 개최 목적

같은 회사, 인물, 산업, 키워드, 날짜나 도시가 겹치는 것만으로는 병합하지 않는다.
예를 들어 기업인 비공개 라운드테이블과 대통령 해외 순방·공식 정상회의는 참석 인물과
지역이 일부 겹쳐도 다른 사건이다.

Solar 호출 실패, 파싱 실패, 후보 목록에 없는 ID 응답은 임의로 합치지 않고
`pending_retry`로 저장해 다음 실행에서 재시도한다.

## 클러스터 갱신과 요약

동일 사건이면 새 기사 임베딩을 반영해 centroid, 기사 수, 마지막 활성 시각과 대표
기사를 갱신한다. 다른 사건이면 새 클러스터를 만든다.

클러스터 배정이 끝난 후 Solar가 포함 기사들을 통합해 다음 값을 저장한다.

- `summary_title`: 사건 전체를 대표하는 제목
- `easy_explanation`: 초보자가 이해하기 쉬운 핵심 설명
- `factual_body`: 여러 기사에서 확인된 사실 중심 통합 본문
- `summary_status`: 요약 성공 여부

감성분류는 대표 기사 제목이 아니라 요약이 성공한 뒤의 `summary_title`을 사용한다.
세부 운영 기준은 [NEWS_CLUSTER_SENTIMENT.md](NEWS_CLUSTER_SENTIMENT.md)를 따른다.

## 모델 선택 검증 결론

BGE-M3는 한국어 뉴스에서 넓은 의미 후보를 찾는 용도로 유지한다. 실제 검증에서 같은
사건 쌍은 다른 사건 쌍보다 높은 유사도를 보였지만, 인물·AI·지역이 겹치는 서로 다른
사건도 후보 하한을 넘을 수 있었다. 이는 임베딩 오류라기보다 후보 검색 모델의 정상적인
특성이다.

따라서 현재 구조는 BGE-M3를 교체하는 대신 다음과 같이 역할을 분리한다.

- BGE-M3: 놓치지 않도록 후보를 넓게 검색
- 24시간 창: 오래된 유사 사건 재결합 가능성 축소
- Solar: 사건 정체성을 비교해 최종 병합 또는 분리

클러스터 품질은 모델 이름만 바꾸기보다 실제 오배정 표본의 정답 라벨을 쌓아
후보 recall과 최종 precision을 함께 평가해야 한다.
