## 회사 기본정보와 기업 일정

- `company_profiles`: 5종목 모두 `company.json` 원본 JSON과 정규화 필드를 저장했다.
- 삼성전자·SK하이닉스·현대차의 `ir_url`은 OpenDART 원본이 빈 값이라 `null`이다.
- `corporate_events`: 142행. 날짜가 원문에 명시되어 파싱된 일정은 87행이다.
- 날짜가 없는 55행은 공시를 일정 후보로 보존하되 날짜를 추측하지 않았다.
- 일정 종류: 배당, 실적, IR, 기준일, 주주총회. 시간·장소·금액도 원문에 명시된 값만 저장한다.

## 정정공시

- 원문에서 `정정대상 공시서류의 최초제출일`을 찾고, 같은 종목·정규화 제목·날짜의 원본이 정확히 하나일 때만 연결했다.
- `supersedes_rcept_no`가 저장된 정정본은 29건이다.
- 원문이 없거나 최초제출일이 없거나 원본을 유일하게 확정할 수 없는 정정 후보 166건은 연결하지 않았다.
- 최신본, 취소·철회 상태 메타데이터를 보존해 향후 RAG에서 `is_latest=true`를 우선할 수 있다.

## 멱등성 검증

같은 run-key와 같은 명령을 재실행했다.

- 고가치 원문 선택: 전 종목 0건
- 주요사항 API 재호출: 0회, 완료된 180건 전부 skip
- 정기보고서 API 재호출: 0회, 완료된 160건 전부 skip
- 재무·구조화·일정 행 증가: 0
- 유일키 중복: disclosures 0, financials 0, structured_disclosures 0, corporate_events 0
- 재실행 중 공시 목록이 33행 늘어난 것은 같은 시각 OpenDART에 새로 게시된 삼성전자 임원·주요주주 보고서 33개의 신규 `rcept_no`이며 중복이 아니다. 이 저우선순위 원문은 내려받지 않았다.

## 남은 제약

1. 한화오션 `20260305000010` 1건은 OpenDART `document.xml`이 `status=014`를 반환한다. `parse_status=unavailable`로 영속화했다.
2. 낮은 우선순위 보유상황 원문은 비용·RAG 노이즈를 줄이기 위해 의도적으로 수집하지 않았다.
3. 정정 후보 166건은 결정론적으로 원본을 확정할 근거가 부족해 연결하지 않았다.
4. 기업 일정 55건은 날짜가 원문에서 명확히 추출되지 않아 날짜를 비워 두었다.
5. 로컬 원본 디렉터리는 DB 백업과 별개이므로 운영 환경에서는 Supabase Storage 또는 영속 볼륨으로 옮겨야 한다.

## RAG 준비 판정

고가치 공시 RAG의 원천 데이터로 사용할 수 있다. 전체 텍스트, 원본 ZIP, 출처 접수번호, 경로, 글자 수, SHA-256, 파싱 상태, 종목, 날짜, 구조화 데이터, 최신 정정본 메타데이터가 준비됐다. 다음 단계에서는 `parse_status=success`, `is_latest=true`를 기본 필터로 삼아 문서 단위가 아닌 섹션 단위 청킹을 수행해야 한다.
